Label-Format (DHL printFormat)
Wichtig: Das Label-Format muss exakt zur Etikettengröße deines Druckers passen, sonst druckt Chrome den Bogen über mehrere Blätter oder skaliert klein in eine Ecke. Für deinen POLONO PL60 (100×150 mm Etiketten) ist 910-300-400 richtig.
Tipp für stillen Druck: Standardmäßig zeigt Chrome einen Druckdialog. Damit das Label ohne Bestätigung direkt auf den Standarddrucker geht, starte Chrome mit dem Flag --kiosk-printing (Verknüpfung anpassen). Andernfalls musst du im Druckdialog nur noch Enter drücken.

Wichtig: Im Druckdialog bei „Weitere Einstellungen“ → Ränder: Keine und Skalierung: 100 % einstellen, sonst druckt Chrome das Label kleiner als das Etikett.

Hinweis: Die Empfänger-ID legst du im DHL Geschäftskundenportal unter „Retoure Online → Empfänger verwalten" an. Die zugehörige Retouren-Adresse (Tommy Hai Nam Nguyen, …) ist dort gespeichert und wird automatisch auf das Retourenlabel gedruckt — du gibst hier nur die Kennung an.
Sales / Discord
Sales-Aggregation: Wenn das Flask-Backend (http://localhost:5000) läuft, teilen sich Web-App und Extension denselben State (sales_state.json) und dieselbe Discord-Nachricht. Andernfalls fällt die Extension automatisch auf lokalen State (chrome.storage) und direkten Discord-Post zurück — kein Setup nötig.
Rechnungs-Empfänger
Pro Empfänger eine eigene Live-Sales-Übersicht in einem eigenen Discord-Channel. Beim Label-Erstellen wählst du, wem dieser Sale zugeordnet wird.
